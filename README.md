- рҹ”Қ **Slither вҖ” Static Analysis** вҖ” Powered by [Slither](https://github.com/crytic/slither), Trail of Bits' industry-standard framework; blazing fast, zero execution required
- рҹ§  **Mythril вҖ” Symbolic Execution** вҖ” Powered by [Mythril](https://github.com/Consensys/mythril) by ConsenSys; explores all possible execution paths to catch deep logic bugs Slither may miss
- рҹ“Ӯ **Bulk Folder Audit** вҖ” Recursively scan an entire project folder and audit every `.sol` file in one click
- рҹ“Ҡ **Visual Statistics** вҖ” Live bar chart showing High / Medium / Low vulnerability distribution after every scan
- рҹ“„ **PDF Report Export** вҖ” Export a clean, professional audit report to any location on your system
- рҹҗі **Docker Isolated** вҖ” Slither runs in `trailofbits/eth-security-toolbox`; Mythril runs in `mythril/myth` вҖ” fully sandboxed
- рҹ–ҘпёҸ **Modern GUI** вҖ” Built with `customtkinter` for a clean, distraction-free dark-mode auditing experience
- вҡЎ **Fast Pipeline** вҖ” From file selection to full dual-engine audit report in seconds

The scanner is a **Python-based dual-engine orchestration layer** that:
1. Accepts `.sol` files or entire folders via GUI
2. Recursively collects all Solidity contracts from the selected path
3. **Engine 1 вҖ” Slither:** Mounts each contract into `trailofbits/eth-security-toolbox`; runs fast static pattern matching
4. **Engine 2 вҖ” Mythril:** Mounts each contract into `mythril/myth`; performs deep symbolic execution to find runtime logic flaws
5. Merges results, renders a severity bar chart + structured findings list in the GUI
6. Exports a clean unified PDF audit report to a user-chosen location

### Step 1 вҖ” Clone the Repository

```bash
git clone https://github.com/onlyrahimsec/Web3-Smart-Contract-Scanner.git
cd Web3-Smart-Contract-Scanner
```

### Step 2 вҖ” Install Python Dependencies

```bash
pip install customtkinter matplotlib reportlab
```

### Step 3 вҖ” Pull the Security Toolbox Images

```bash
# Slither engine (Trail of Bits)
docker pull trailofbits/eth-security-toolbox

# Mythril engine (ConsenSys)
docker pull mythril/myth
```

> рҹ’Ў These images bundle all required Solidity compilers and analysis tools. Pull once, audit forever.

### Step 4 вҖ” Launch the Scanner

```bash
python gui.py
```

## рҹӣЎ Vulnerabilities Detected

| Severity | Vulnerability Type | Impact | Fix |
|----------|--------------------|--------|-----|
| рҹ”ҙ **HIGH** | `reentrancy-eth` | Funds can be drained repeatedly via re-entry | Use `ReentrancyGuard` |
| рҹ”ҙ **HIGH** | `suicidal` | Anyone can destroy the contract | Add `onlyOwner` checks |
| рҹҹ  **MEDIUM** | `unchecked-send` | Failed transfers go undetected | Strictly check return values |
| рҹҹЎ **LOW** | `missing-zero-check` | Zero-address assignments cause silent bugs | Validate all address inputs |
| рҹ”ө **INFO** | `solc-version` | Outdated compiler may have known bugs | Use `^0.8.0` or later |
| рҹ”ө **INFO** | `low-level-calls` | Risky use of `.call()` / `.send()` | Prefer `transfer()` or checks-effects-interactions |
| рҹ”ө **INFO** | `naming-convention` | Non-standard variable/function naming | Follow Solidity style guide |

## вҡҷпёҸ Configuration

| Option | Default | Description |
|--------|---------|-------------|
| Engine 1 | `Slither` | Static analysis вҖ” fast pattern-based detection |
| Engine 2 | `Mythril` | Symbolic execution вҖ” deep runtime logic analysis |
